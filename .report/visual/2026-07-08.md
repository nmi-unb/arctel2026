## 3. Nav no desktop

**Não é `fixed` nem `sticky`.** É `position: static` (padrão) — rola junto com o conteúdo.

CSS responsável: `assets/css/layout/header.css:1-10`
```css
.site-header {
  display: flex;
  align-items: center;
  justify-content: space-between;
  flex-wrap: wrap;
  gap: 24px;
  padding: 24px var(--grid-margin);
  max-width: var(--container-max-width);
  margin: 0 auto;
}
```
Sem `position`, sem `top`, sem `z-index` no `.site-header` em si. Os links (`.site-nav`) também são só `display:flex` normal (`header.css:28-33`).

Busquei `position: fixed|sticky` em todo `assets/css` — só aparece dentro do bloco do menu hambúrguer/drawer mobile (ver item 4). Nenhum breakpoint torna o header fixo no desktop.

## 4. Nav no mobile

Muda bastante, em 2 breakpoints:

**`@media (max-width: 900px)`** (`header.css:109-221`):
- `.menu-toggle` (botão hambúrguer) vira `display: block` — ele já é `position: fixed; top:24px; right:24px; z-index:1200` incondicionalmente (`header.css:44-46`), então mesmo esse breakpoint só controla visibilidade, a fixação já valia antes.
- `.site-nav` (os 6 links) vira um **drawer lateral**: `position: fixed; top:0; right:0; width:min(78vw,340px); height:100vh`, escondido fora da tela via `transform: translateX(100%)`, abre com `.site-header.menu-is-open .site-nav { transform: translateX(0) }`.
- `.menu-backdrop` aparece (`position:fixed; inset:0; background: rgba(0,0,0,.18)`), visível só quando `.menu-is-open`.
- Links dentro do drawer perdem visual de pill (sem borda, sem radius, texto à direita, sublinhado verde no ativo).
- `body.menu-open { overflow:hidden }` trava o scroll do fundo enquanto o drawer tá aberto.

**`@media (max-width: 480px)`** (`header.css:115-140` e `223-228`):
- Botão encolhe (52px→48px), reposiciona (`top/right` 24px→18px).
- Drawer fica mais largo em proporção (`width:82vw` em vez de `min(78vw,340px)`) e com mais padding-top.

Também tem `home.css:5-9`: em `max-width:900px`, `.site-header { justify-content:center; text-align:center }` — só recentraliza logo+botão, não mexe em fixação.

**Resumo da diferença hoje:**
- Desktop: nav é uma linha normal dentro do header, tudo `static`, some da tela ao rolar (porque o header inteiro rola).
- Mobile (≤900px): nav vira gaveta lateral fixa que só aparece por cima do conteúdo quando o hambúrguer (esse sim sempre fixo) é clicado. O header com logo continua `static` (rola normal), só o botão hambúrguer e o drawer aberto são fixos.

## 5. Botão "voltar ao topo" / scroll listener existente

**Não existe nenhum botão "voltar ao topo"** em HTML, CSS ou JS.

Scroll listener que já existe (`assets/js/menu.js:53-64`):
```js
function handleMenuButtonScroll() {
  const hasScrolled = window.scrollY > 40;
  if (hasScrolled) {
    menuToggle.classList.add("is-scrolled");
  } else {
    menuToggle.classList.remove("is-scrolled");
  }
}
window.addEventListener("scroll", handleMenuButtonScroll);
```
Serve só pra baixar a opacidade do botão hambúrguer (`.menu-toggle.is-scrolled { opacity:0.4 }`), não tem relação com voltar ao topo. Pra implementar o botão flutuante, dá pra reaproveitar o padrão (listener de `scroll` + toggle de classe), mas precisa de um listener/elemento novo — não dá pra grudar na função acima sem misturar responsabilidades.

Único outro rastro de "scroll" no projeto: `scroll-behavior: smooth` em `assets/css/base/reset.css:11` (no `html`) — já garante que scroll-to-section (via `href="#id"`) sai suave, de graça, sem JS. Bom pra reaproveitar no botão voltar ao topo também (`window.scrollTo({top:0, behavior:'smooth'})` nem precisa do `behavior` explícito já que o `html` já tem `scroll-behavior:smooth` — mas usar explícito não conflita).

## 6. Padrão visual das seções "em desenvolvimento"

Único exemplo hoje: Cronograma (`index.html:182-185`).

```html
<section class="section-cronograma container">
  <h2 class="section-title section-title--dark">Cronograma</h2>
  <div class="section-cronograma__placeholder">Em Desenvolvimento</div>
</section>
```

**Título** (`.section-title.section-title--dark`, `typography.css:14-23`):
- `font-size: var(--text-display)` (40px)
- `font-weight: 800`
- cor: `.section-title` base é `var(--verde-600)`, mas `--dark` sobrescreve pra `var(--color-black)`
- `margin-bottom: 24px`

**Retângulo placeholder** (`.section-cronograma__placeholder`, `sections.css:46-67`):
- `background-color: var(--color-gray-200)` (#d9d9d9)
- `border-radius: var(--radius-sm)` (4px) — **sem borda** (`border` não é setado)
- `min-height: 320px` (cai pra 200px em ≤480px)
- `display:flex; align-items:center; justify-content:center` (texto centralizado nos 2 eixos)
- texto: `font-size: var(--text-display)` (40px), `font-weight:800`, `text-align:center`, `overflow-wrap:break-word`
- em `≤480px`: `font-size` cai pra `var(--text-xl)` (24px) e ganha `padding:16px` — evita a palavra "Desenvolvimento" estourar a largura

Pra reaproveitar exatamente esse padrão numa seção nova (Noticias/Materiais/Contato), é só repetir a mesma dupla `h2.section-title` (+ `--dark` se quiser título preto em vez de verde) e `div` com essa classe de placeholder — nenhuma cor/fonte nova precisa ser inventada.
