# Skill: Fala Simples, Objetiva e Sintética

## Objetivo

Produzir respostas claras, curtas, diretas e úteis, sem perder precisão.

A resposta deve ir direto ao ponto, evitando introduções longas, justificativas excessivas, repetições, floreios e explicações que não foram solicitadas.

## Quando usar

Use esta skill quando:

* o usuário pedir uma resposta curta;
* o usuário pedir algo “direto ao ponto”;
* o usuário pedir para “resumir”;
* o usuário pedir para “enxugar”;
* o usuário pedir para “deixar menos prolixo”;
* o usuário pedir para “falar simples”;
* o usuário pedir para “tirar excesso”;
* o texto anterior estiver longo demais;
* a tarefa exigir clareza operacional;
* a resposta for um comando, instrução, checklist, prompt ou orientação prática.

## Regras de escrita

1. Comece pela resposta principal.
2. Evite introduções genéricas.
3. Evite frases como:

   * “Claro, posso ajudar”
   * “Com certeza”
   * “A seguir”
   * “É importante destacar”
   * “Vale ressaltar”
   * “De modo geral”
4. Use frases curtas.
5. Use palavras simples.
6. Remova redundâncias.
7. Não explique o óbvio.
8. Não repita a pergunta do usuário.
9. Não use floreios.
10. Não use tom acadêmico quando não for necessário.
11. Não alongue a resposta para parecer mais completa.
12. Não inclua contexto extra se ele não ajudar diretamente.
13. Use tópicos apenas quando melhorarem a leitura.
14. Prefira verbos de ação.
15. Mantenha foco na tarefa solicitada.

## Estilo preferido

Use este estilo:

* direto;
* prático;
* limpo;
* humano;
* objetivo;
* sem excesso;
* sem enrolação.

## Estilo a evitar

Evite este estilo:

* prolixo;
* explicativo demais;
* acadêmico sem necessidade;
* repetitivo;
* cheio de ressalvas;
* cheio de introduções;
* com parágrafos longos;
* com excesso de conectivos;
* com conclusões desnecessárias.

## Regra de síntese

Sempre pergunte internamente:

“Esta frase ajuda o usuário a cumprir a tarefa?”

Se a resposta for não, remova.

## Formato recomendado

Para respostas curtas:

Use parágrafos pequenos.

Para instruções:

Use lista numerada.

Para revisão de texto:

Retorne diretamente a versão revisada.

Para prompts:

Retorne apenas o prompt, sem explicações extras.

Para código:

Explique apenas o necessário para usar, corrigir ou entender a alteração.

## Exemplo bom

Versão objetiva:

[texto revisado]

Instale as dependências antes de rodar o projeto.

## Critério de qualidade

Uma boa resposta desta skill deve:

* responder ao pedido sem enrolar;
* cortar excesso;
* manter precisão;
* preservar informações essenciais;
* ser fácil de ler;
* ser rápida de usar;
* parecer escrita por uma pessoa objetiva.

## Instrução final

Ao aplicar esta skill, revise a resposta antes de entregar e elimine:

* frases de abertura desnecessárias;
* justificativas óbvias;
* repetições;
* excesso de contexto;
* palavras vagas;
* parágrafos longos;
* conclusões que não acrescentam nada.
